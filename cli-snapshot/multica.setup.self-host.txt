Configures the CLI to connect to a self-hosted Multica server.

By default, connects to http://localhost:8080 (backend) and http://localhost:3000 (frontend).
Use --server-url and --app-url to specify a custom server (e.g. an on-premise deployment).

If you run this command from a different machine than the server, also pass
--callback-host <FQDN-or-IP-the-browser-can-reach-back-to-this-machine-on> so
the OAuth login flow can return the token to the CLI.

Examples:
  multica setup self-host
  multica setup self-host --server-url https://api.internal.co --app-url https://app.internal.co
  multica setup self-host --port 9090 --frontend-port 4000

USAGE
  multica setup self-host [flags]

FLAGS
      --app-url string         Frontend app URL (e.g. https://app.internal.co)
      --callback-host string   Host the OAuth callback URL points at (auto-detected when empty). Use this for reverse-proxy / FQDN setups.
      --frontend-port int      Frontend port (used when --app-url is not set) (default 3000)
  -h, --help                   help for self-host
      --port int               Backend server port (used when --server-url is not set) (default 8080)
      --server-url string      Backend server URL (e.g. https://api.internal.co)

INHERITED FLAGS
  --help   Show help for command
